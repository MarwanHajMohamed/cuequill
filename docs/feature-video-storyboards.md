# Feature tutorial storyboards — marketing

Short, hook-first clips that **show the feature being used**, built for the
landing/features pages (and re-cuttable for social). Each is designed to
autoplay muted with on-screen captions, so the **caption** carries the
message and the **voiceover (VO)** is optional.

## House style
- **Length:** 18–28s each. First 2s must show value, not a logo.
- **Format:** screen capture at 1280×800 (desktop) or 1080×1920 (a phone
  cut for social). Dark theme by default; grab one light-theme take of the
  calendar for variety.
- **Pacing:** one idea per shot, ~3–5s each. Cursor moves deliberately;
  pause ~0.8s after each click so the result reads.
- **Captions:** bottom third, sentence case, ≤7 words. Teal highlight on
  the key noun (matches the site accent `#5eead4`).
- **End card (2s):** Cuequill wordmark + "Start free · cuequill.app".
- **Seed data:** a demo account with ~1 month of mixed wins/losses across
  3–4 symbols and 2–3 strategies, so every screen looks alive.

## Suggested rollout
- **Landing hero loop:** a silent 10s supercut of shots from #1, #2, #4.
- **Per-feature section:** drop the matching clip beside each feature block
  on `/features` (replaces or augments the static preview mockups).
- **Social:** the 9:16 cuts of #2 (Quill AI) and #5 (Log a trade) travel
  best — they show a result fast.

## Recording checklist
- Seed the demo account first; never use real account data or a real
  Flex token on screen.
- Hide the macOS/Windows clock and notifications.
- Record at 60fps; export at 30fps for smaller files.
- Keep a clean cursor; consider a click-highlight overlay.
- Export `.mp4` (H.264) for the site + a `.webm` fallback.
